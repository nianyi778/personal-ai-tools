# Gotchas（已知陷阱）

## Jira API 相关

### 1. 旧搜索 API 已废弃
**问题**: `GET /rest/api/3/search?jql=...` 返回错误 "请求的 API 已被移除"
**解决**: 使用 `POST /rest/api/3/search/jql`，JSON body 传参

### 2. rtk 过滤 curl JSON 输出
**问题**: rtk 安全机制将 JSON 字段值替换为类型标识（string, int, bool），直接解析失败
**解决**: 必须 `-o /tmp/xxx.json` 存文件，再用 python json.load 解析

### 3. 附件下载需要 -L
**问题**: Jira 附件 content URL 会 302 重定向，不加 `-L` 只下载到 63 字节的重定向页面
**解决**: `curl -s -L -u "$EMAIL:$TOKEN" "$URL" -o /tmp/file.ext`

### 4. @mention 渲染为纯文本
**问题**: 评论中写 `@徐哲` 不会渲染为可点击链接
**解决**: 使用 ADF 格式的 mention 节点，传入 accountId（见 templates.md）

### 5. Jira text search 对短关键词报错
**问题**: `text ~ "T-0125"` 可能因连字符导致搜索失败
**解决**: 用 `project=AT AND assignee=currentUser()` 搜全部，python 过滤 T- 编号

## Google Sheet 相关

### 6. 行号计算因跳号出错
**问题**: T-0069, T-0126, T-0187, T-0224 不存在，编号直接计算行号会偏移
**解决**: 始终通过 B 列实际内容查找确认行号，不盲信计算

### 7. Sheet ID 不固定
**问题**: 缺陷管理表可能被复制/新建，ID 会变化
**解决**: 每次 session 用 `searchDriveFiles(query="不具合管理")` 搜索，取 modifiedTime 最新的

### 8. 列头区域有合并单元格
**问题**: 行 4-5 是分组头（■基本情報 等），行 6 才是字段头，直接读 A1:AH1 可能为空
**解决**: 读 A6:AH6 获取实际列头

## 分析流程相关

### 9. 仅凭评论区判断
**问题**: 评论区的判断可能不准确或是临时方案
**解决**: 必须查看代码验证，眼见为实。评论只作参考

### 10. 忽略 Excel 附件
**问题**: 只看图片附件，遗漏 Excel 管理表中的关键信息（内容詳細、期待値、発生方法）
**解决**: 全部附件都下载。Excel 用 openpyxl 解析，重点看 `内容(詳細)` 和 `期待値`

### 11. 图片下载失败不报错
**问题**: 用了错误的 attachment ID 或缺少 -L，下载到空文件但不报错
**解决**: 下载后检查文件大小（`ls -la`），小于 1KB 的大概率是失败

### 12. 跳步不等确认直接回填
**问题**: 为求速度跳过 Step 2e（等用户确认），直接批量回填
**解决**: **铁律**：必须展示分析结论，等用户说 OK 后才回填

### 13. AF 列内容格式不统一
**问题**: 每次临时编写格式，导致 AF 列内容风格不一致
**解决**: 固定三行模板 `詳細発生原因/詳細対応方法/詳細影響範囲`

### 14. Jira 描述与 Sheet 概要不一致
**问题**: Jira ticket 的 summary 可能被修改（如加上 "🔵直接改"），与 Sheet D 列概要不同
**解决**: 以 Sheet D 列概要为正式描述，Jira 做参考。如有重大出入需在 AF 中注明

### 15. 重复单互相引用造成循环
**问题**: T-0178 说"重複 T-0197"，T-0197 说"重複 T-0178"，无法判断哪个是原单
**解决**: 看起票时间，先起的是原单。两个都标记，AF 写清引用关系

### 16. 回填后不验证
**问题**: batchWrite 成功不代表值正确（可能行号算错写到了别的行）
**解决**: 回填后必须 `readSpreadsheet` 验证值匹配。不验证=不闭环
